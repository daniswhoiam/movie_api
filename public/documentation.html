Documentation

This is the documentation for the API of the "myFlix" application, made by @daniswhoiam.

Introduction

This API will be dealing with requests to the database of the "myFlix" application. It will serve data about movies, genres, directors etc. Moreover, it will deal with user interactions with the database, e.g. updating their personal information.

Request URL HTTP Method Query Parameters Request body data format Response body data format
Return a list of all movies to the user /movies GET No query parameters. No request body needed.

An array holding all the movies from the database in JSON format:

[{
"Title": "sampleTitle",
"Description": "...",
"ReleaseYear": "sampleYear"
},
{
"Title": "sampleTitle",
"Description": "...",
"ReleaseYear": "sampleYear"
},
...]
Return data about a single movie by title to the user /movies/:title GET title No request body needed.

A JSON object holding data about the requested movie:

{
"Title": "sampleTitle",
"Description": "...",
"ReleaseYear": "sampleYear"
}

Return data about a genre by title /genres/:title GET name No request body needed. A JSON object with the genre data:
{
"Name": "Science Fiction",
"Description": "Science Fiction (or sci-fi) is a film genre that uses speculative, fictional science-based depictions of phenomena that are not fully accepted by mainstream science, such as extraterrestrial lifeforms, alien worlds, extrasensory perception and time travel, along with futuristic elements such as spacecraft, robots, cyborgs, interstellar travel or other technologies. Science fiction films have often been used to focus on political or social issues, and to explore philosophical issues like the human condition."
}
Return data about a director by name /directors/:name GET name No request body needed. A JSON object with the genre data:
{
"Name": "Steven Allan Spielberg",
"Bio": "Steven Allan Spielberg is an American film director, producer and screenwriter. He began his career in the New Hollywood era, and is one of the most commercially successful directors in history.",
"Birth": "1946-12-18T00:00:00.000Z",
"Death": null
}
Allow new users to register /users POST No query parameters.

A JSON object with user data:

{
"username": "test999",
"password": "sdf54sd65g6sd8fe48s",
"email": "test999@test.de",
"birth": "2000-01-01"
}

A JSON object with the data of the newly created user:
{
"FavoriteMovies": [],
"_id": "60697ba2e27b431378879e0e",
"Username": "test999",
"Password": "sdf54sd65g6sd8fe48s",
"Email": "test999@test.de",
"Birth": "2000-01-01T00:00:00.000Z",
"__v": 0
}
Allow users to update their data /users/:username PUT username

A JSON object with user data:

{
"username": "test999",
"password": "sdf54sd65g6sd8fe48s",
"email": "test999@test.de",
"birth": "1998-02-01"
}

It is enough to include one information to update, e.g. only username.

A JSON object with the data of the updated user data:
{
"FavoriteMovies": [],
"_id": "60697ba2e27b431378879e0e",
"Username": "test999",
"Password": "sdf54sd65g6sd8fe48s",
"Email": "test999@test.de",
"Birth": "1998-02-01T00:00:00.000Z",
"__v": 0
}
Allow users to add a movie to their list of favorites /users/:username/movies/:movieID PATCH username, movieID No request body needed. A JSON object with the data of the updated user data:
{
"FavoriteMovies": ["606616428cf62e963ea9b872"],
"_id": "60697ba2e27b431378879e0e",
"Username": "test999",
"Password": "sdf54sd65g6sd8fe48s",
"Email": "test999@test.de",
"Birth": "1998-02-01T00:00:00.000Z",
"__v": 0
}
Allow users to remove a movie from their list of favorites /users/:username/movies/:movieID DELETE username, movieID No request body needed. A JSON object with the data of the updated user data:
{
"FavoriteMovies": [],
"_id": "60697ba2e27b431378879e0e",
"Username": "test999",
"Password": "sdf54sd65g6sd8fe48s",
"Email": "test999@test.de",
"Birth": "1998-02-01T00:00:00.000Z",
"__v": 0
}
Allow existing users to deregister. /users/:username DELETE username No request body needed. Plain text informing about the success of the request.
Example: "test999 was deleted."
Allow to get user information. /users/:username GET username No request body needed. A JSON object with the user data:
{
"FavoriteMovies": [],
"_id": "60697ba2e27b431378879e0e",
"Username": "test999",
"Password": "sdf54sd65g6sd8fe48s",
"Email": "test999@test.de",
"Birth": "1998-02-01T00:00:00.000Z",
"__v": 0
}
Allow to get the user's favorite movies. /users/:username/favoritemovies GET username No request body needed. An array containing the favorite movies. Example:
["606616428cf62e963ea9b872"]
