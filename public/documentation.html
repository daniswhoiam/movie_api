Documentation

This is the documentation for the API of the "myFlix" application, made by @daniswhoiam.

Introduction

This API will be dealing with requests to the database of the "myFlix" application. It will serve data about movies, genres, directors etc. Moreover, it will deal with user interactions with the database, e.g. updating their personal information.

Request URL HTTP Method Query Parameters Request body data format Response body data format
Return a list of all movies to the user /movies GET No query parameters. No request body needed.

A JSON object holding data about all movies in the database:

[{
"Title": "sampleTitle",
"ReleaseYear": "sampleYear"
},
...]
Return data about a single movie by title to the user /movies/:title GET title No request body needed.

A JSON object holding data about the requested movie:

{
"Title": "sampleTitle",
"ReleaseYear": "sampleYear"
}

Return data about a genre by title /genres/:name GET name No request body needed. In the future: A JSON object with the genre data. Currently plain text informing about the success of the request.
Example: "Successfully retrieved data about genre: sampleTitle"
Return data about a director by name /directors/:name GET name No request body needed. In the future: A JSON object with the genre data. Currently plain text informing about the success of the request.
Example: "Successfully retrieved data about director: sampleName"
Allow new users to register /users POST No query parameters.

A JSON object with user data:

{
"Username": "sampleUsername",
"Password": "samplePassword",
"Email": "sample@email.com",
"Birth": "1998-02-01"
}

In the future: A JSON object with the genre data. Currently plain text informing about the success of the request.
Example: "Successfully created user."
Allow users to update their username /users/:userID/:username PUT userID, username No request body needed. Plain text informing about the success of the request.
Example: "The username of user #1 was successfully updated."
Allow users to add a movie to their list of favorites /users/:userID/favorites PATCH userID

A JSON object holding data about the movie:

{
"Title": "sampleTitle",
"ReleaseYear": "sampleYear",
"movieID": "sampleID" }

Plain text informing about the success of the request.
Example: "The movie has been successfully added to user #1's favorites list."
Allow users to remove a movie from their list of favorites /users/:userID/favorites/:movieID PATCH userID, movieID No request body needed. Plain text informing about the success of the request.
Example: "The movie #222 was removed from user #1's favorites list."
Allow existing users to deregister. /users/:userID DELETE userID No request body needed. Plain text informing about the success of the request.
Example: "User #1 was removed from the system."
